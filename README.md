# 럭키★오락실 (Lucky Arcade)

RisuAI 계열 봇카드(.charx / .png / .risum)를 드롭하면, 카드 안에 이미 있는 로어북·인물·이미지·룰 테이블을
재료로 **LLM 호출 0회, 100% 결정론 엔진**으로 돌아가는 미니게임을 자동으로 여는 로컬 웹앱.

> 봇카드는 대화 상대인 동시에, 세계관·등장인물·이미지·규칙이 들어 있는 로컬 게임 카트리지다.

## 핵심 원칙

1. **무LLM** — 게임 규칙·판정·진행·보상은 전부 코드가 담당한다. API 키·네트워크가 필요 없다.
2. **카드에 없는 설정을 발명하지 않는다** — 화면의 모든 사실은 카드 원문 또는 투명한 구조 유래 공식에서 나오며,
   합성 수치에는 출처(`derived`)를 표시한다.
3. **근거가 있을 때만 게임이 열린다** — 재료 검사기가 카드마다 가능한 캐비닛(게임)만 활성화한다.
   억지 게임·빈 화면을 만들지 않는다.
4. **카드 스크립트는 실행하지 않는다** — 카드 내장 Lua/JS는 문자열 자료로만 취급한다.
   정규식은 실행 예산과 안전 검사를 거친다.
5. **로컬 전용** — 카드 분석·플레이·저장은 전부 브라우저 안에서 끝난다. 카드 원본과 에셋은 서버로 보내지 않는다.
6. **가벼운 결정론** — 시드 난수·입력 기록·결과 해시·게임 버전·카드 지문으로 같은 판을 누구나 재현할 수 있다.

## 라이선스와 콘텐츠 권리 (중요)

- **프로그램**: [GPL-3.0-or-later](./LICENSE). 이 프로그램은 GPL 조건에 따라 자유롭게 복사·수정·재배포할 수 있다.
- **사용자가 불러온 봇카드·이미지·음악·텍스트**: 이 프로그램의 라이선스에 **포함되지 않는다**.
  각 카드의 권리는 각 권리자에게 있다.
- 이 앱은 카드 원본이나 변환 결과를 재수출·재배포하는 기능을 만들지 않는다.
- 도전 코드·리플레이 등 공유물에는 **카드 지문·시드·점수·입력 기록만** 담기며,
  카드 원문·이미지·로어 본문은 포함되지 않는다.

## 관련 프로젝트

- [lucky-simulator](https://github.com/dootaang/lucky-simulator) — 같은 카드 생태계를 다루는 자매 프로젝트
  (Risu 호환 플레이어·시뮬 변환기). 럭키★오락실은 그중 순수 함수(카드 파서·로어 활성화기)만
  출처를 기록하고 선별 이식한다. 외부 코드 출처는 [docs/THIRD_PARTY_PROVENANCE.md](./docs/THIRD_PARTY_PROVENANCE.md).

## 현재 단계

- TypeScript 7 strict 기반 pnpm 모노레포와 React 19/Vite 8/Tailwind 4 웹 셸
- 전체 파일을 메모리에 올리지 않는 `BinarySource` 카드 파서
- Worker에서 실행되는 로컬 카드 분석과 IndexedDB 카드 보관함
- 로어 그래프·NPC 스프라이트·경제표·캐비닛 적합도 보고서
- 플레이 가능한 1호 캐비닛 `로어 회로`(결정론 시드·행동 영수증·새로고침 복구)
- 데스크톱·모바일 브라우저 회귀검사와 초기 JavaScript 200 KiB gzip 예산

```bash
pnpm install
pnpm check
pnpm test:e2e
pnpm report -- "C:\\path\\to\\card.charx"
pnpm dev
```

보고서는 로컬 `reports/`에 생성되며 Git에 포함되지 않는다. 카드 원문과 에셋 바이트는 보고서에 기록하지 않는다.
