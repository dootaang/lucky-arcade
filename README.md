# 럭키★오락실 (Lucky Arcade)

RisuAI 사용자가 API 응답을 기다리는 동안, 봇카드 안에 이미 있는 로어북·인물·이미지·룰 테이블로 만든
게임을 즐기는 **LLM 호출 0회, 100% 결정론 오락실**. 운영자가 준비한 내장 게임은 카드 파일 없이 바로
실행하고, 원하면 개인 봇카드를 넣어 그 카드에 맞는 게임도 열 수 있다.

> 봇카드는 대화 상대인 동시에, 세계관·등장인물·이미지·규칙이 들어 있는 로컬 게임 카트리지다.
> 소녀전선: 잔불 작전은 제품 전체가 아니라 오락실의 첫 대형 캐비닛이자 기술 검증작이다.

## 내장 게임

- **소녀전선 최애 월드컵** — 12명 중 오늘의 최애를 정하는 1~3분 개봉식.
- **작전 암호 기억** — 차례로 나타난 인물을 같은 순서로 선택하는 5라운드 기억 게임.
- **럭키★더비 엔진 실험장** — 같은 8인 경주를 Phaser·melonJS·Excalibur·LittleJS로 재생해 다음 게임의 표현 엔진을 실제 기기에서 고르는 비교 캐비닛.
- **소녀전선: 잔불 작전** — 12명 중 6명을 편성하고 7개 작전 구간을 돌파하는 결정론 전술 오토배틀 로그라이트. 카드 업로드 없이 로비에서 바로 시작할 수 있으며 모든 진행은 브라우저에 저장된다.

## 핵심 원칙

1. **무LLM** — 게임 규칙·판정·진행·보상은 전부 코드가 담당한다. API 키·네트워크가 필요 없다.
2. **카드에 없는 설정을 발명하지 않는다** — 화면의 모든 사실은 카드 원문 또는 투명한 구조 유래 공식에서 나오며,
   합성 수치에는 출처(`derived`)를 표시한다.
3. **근거가 있을 때만 게임이 열린다** — 재료 검사기가 카드마다 가능한 캐비닛(게임)만 활성화한다.
   억지 게임·빈 화면을 만들지 않는다.
4. **카드 스크립트는 실행하지 않는다** — 카드 내장 Lua/JS는 문자열 자료로만 취급한다.
   정규식은 실행 예산과 안전 검사를 거친다.
5. **로컬 전용** — 카드 분석·플레이·저장은 전부 브라우저 안에서 끝난다. 카드 원본과 에셋은 서버로 보내지 않는다.
6. **가벼운 결정론** — 시드 난수·입력 기록·결과 해시·게임 버전·카드 지문으로 같은 판을 누구나 재현할 수 있다.

## 라이선스와 콘텐츠 권리 (중요)

- **프로그램**: [GPL-3.0-or-later](./LICENSE). 이 프로그램은 GPL 조건에 따라 자유롭게 복사·수정·재배포할 수 있다.
- **사용자가 불러온 봇카드·이미지·음악·텍스트**: 이 프로그램의 라이선스에 **포함되지 않는다**.
  각 카드의 권리는 각 권리자에게 있다.
- 이 앱은 카드 원본이나 변환 결과를 재수출·재배포하는 기능을 만들지 않는다.
- 도전 코드·리플레이 등 공유물에는 **카드 지문·시드·점수·입력 기록만** 담기며,
  카드 원문·이미지·로어 본문은 포함되지 않는다.

## 관련 프로젝트

- [lucky-simulator](https://github.com/dootaang/lucky-simulator) — 같은 카드 생태계를 다루는 자매 프로젝트
  (Risu 호환 플레이어·시뮬 변환기). 럭키★오락실은 그중 순수 함수(카드 파서·로어 활성화기)만
  출처를 기록하고 선별 이식한다. 외부 코드 출처는 [docs/THIRD_PARTY_PROVENANCE.md](./docs/THIRD_PARTY_PROVENANCE.md).

## 현재 단계

- TypeScript 7 strict 기반 pnpm 모노레포와 React 19/Vite 8/Tailwind 4 웹 셸
- 전체 파일을 메모리에 올리지 않는 `BinarySource` 카드 파서
- Worker에서 실행되는 로컬 카드 분석과 IndexedDB 카드 보관함
- 로어 그래프·NPC 스프라이트·경제표·캐비닛 적합도 보고서
- 개봉식 `최애 월드컵`(8~16강·부전승·오늘의 16명)과 첫 게임 `카드 복구반`(초상·표정·이름 변조 찾기)
- 실험실 캐비닛 `로어 회로`(결정론 시드·행동 영수증·새로고침 복구)
- 선택한 초상만 Worker에서 만드는 지연 썸네일과 32장 URL 캐시
- 데스크톱·모바일 브라우저 회귀검사와 초기 JavaScript 200 KiB gzip 예산(현재 약 105 KiB)
- 카드 없는 `바로 놀기`, 최근 진행 `이어하기`, 선택형 `내 카드로 놀기`로 분리된 오락실 로비
- 한 내장 콘텐츠팩으로 월드컵·기억 게임·장기 작전을 여는 `BuiltInContentPack` 계약
- 같은 결정론 경주를 네 2D 엔진으로 재생하고 첫 화면·프레임 비용을 자동 비교하는 `ArcadeRuntimeAdapter` 실험장
- 《테메로세: 여백》 D1용 실제 MIME 스니핑·명시적 SFW 허용 목록·장면별 WebP 콘텐츠팩
- 제품 방향은 [ROADMAP](./docs/ROADMAP.md), 유지할 맥락은 [PRODUCT-MEMORY](./docs/PRODUCT-MEMORY.md),
  다음 구현 순서는 [BACKLOG](./docs/BACKLOG.md)를 따른다.

```bash
pnpm install
pnpm check
pnpm test:e2e
pnpm report -- "C:\\path\\to\\card.charx"
pnpm measure:npc -- "C:\\path\\to\\card.charx"
pnpm content:validate -- "C:\\path\\to\\built-in-pack.json"
pnpm content:temerosa:audit -- "apps\\web\\public\\content\\temerosa-margin\\0.1.0\\manifest.json"
pnpm dev
```

보고서는 로컬 `reports/`에 생성되며 Git에 포함되지 않는다. 카드 원문과 에셋 바이트는 보고서에 기록하지 않는다.
