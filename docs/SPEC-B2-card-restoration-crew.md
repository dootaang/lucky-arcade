# SPEC-B2 — 카드 복구반 캐비닛 (첫 게임 실험)

> 발주: 지휘자, 2026-07-23 (오너 전체 승인). 구현: Codex. **SPEC-B1 완료 후 착수.**
> 상위 문서: [README](../README.md) · [ROADMAP "제품 가설 v2"](./ROADMAP.md).
> 손맛 정의: 한국 이용자에게 익숙한 것은 긴 설정 퀴즈가 아니라 **"이상한 곳을 눌러라"**다.

## 목표

원본 카드 데이터를 정답지로 삼아 엔진이 기계적으로 오류를 만들고, 플레이어가 제한 시간 안에
잘못된 부분을 찾아내는 3~5문제 1판(총 1~3분)의 게임을 만든다. 이 게임의 변조기·출제기·판정기는
이후 럭키★아카이브의 사건 시스템으로 승격될 기반이다.

## 1. 작업 범위 (건드릴 파일)

### B2-a. 변조기 + 출제기 (packages/extract 신설 모듈, 순수 함수)

- **1차 변조 레퍼토리(쉬운 것만 — 정답이 이미지·이름 수준에서 확실한 것)**:
  1. 초상화 교체 — NPC 이름표는 그대로, 초상화만 다른 인물 것으로
  2. 표정 끼워넣기 — 같은 인물 스프라이트 묶음에 다른 인물 1장 삽입
  3. 이름 교체 — 두 NPC의 이름표 맞바꾸기
  4. 숫자 변조 — 카드에 명시된 수치표(가격·능력치)가 있을 때만, 한 칸을 다른 실값과 교체
- **로어 의미 변조(문장 변조)는 이번 범위에서 제외** — 2차 레퍼토리로 백로그.
- **건전성 검증기(필수)**: 생성된 문제마다 ①정답이 유일한가 ②변조 결과가 카드 안의 다른
  실제 사실과 우연히 일치하지 않는가 ③사용 에셋의 confidence가 기준 이상인가를 검사하고,
  하나라도 실패하면 그 문제를 폐기한다. 폐기 사유는 진단에 기록.
- 출제는 시드 결정론: 같은 시드 → 같은 문제 세트. 문제 수는 재료량에 따라 3~5개.

### B2-b. 복구반 캐비닛 (cabinets/restoration-crew — core/react 분리)

- `core`: 판 상태 리듀서 — 문제 제시→탭 판정(정답/오답 즉시 확정)→다음 문제→판 종료.
  제한 시간(문제당 기본 15초)·연속 정답 콤보·실수 허용 횟수·등급 산정(S/A/B/C)을
  전부 core가 소유. 입력 기록·결과 해시는 `packages/engine` 재사용.
- `react`: 한 화면 = 한 문제. 이상한 곳을 직접 탭 → 즉시 정답 공개(원본 vs 변조 대조 표시,
  원본 출처 명시). 판 종료 화면은 점수보다 **"복구한 인물들"**을 먼저 보여준다.
  즉시 재도전 버튼(같은 카드 새 시드)과 "다른 카드 넣기" 유도.
- 재료 관문: cabinet-sdk 매니페스트에 선언 — 신뢰도 기준을 넘는 NPC 초상 4명 이상 또는
  검증 가능한 수치표 존재. 미달 시 비활성(억지 출제 금지).
- 개봉식 연결: SPEC-B1의 폴백 2번 슬롯을 복구반으로 교체(월드컵 관문 미달 카드의 첫 체험).
  월드컵 우승 화면의 "이 인물과 함께 기록을 복구하시겠습니까?" 훅으로도 진입.

### B2-c. 검증

- 단위: 변조 4종 각각의 생성·건전성 검증기(유일해·우연 일치 폐기)·시드 재현·등급 산정.
- E2E: 합성 카드로 개봉식→복구반 1판 완주→재도전 여정.
- **실카드 5종 기술 검증(합의된 4단계)**: 유형이 다른 실보유 카드 5종(이미지형·로어형·혼합형·
  수치표형·빈약형)에서 ①NPC 그룹화 정확도(오합침 발견 시 기록) ②문제 생성 건전성(폐기율 포함)
  ③관문 차단 정확도를 표로 보고. 빈약형 카드에서 캐비닛이 정직하게 닫히는지 포함.

## 2. 금지사항

- LLM·네트워크 전송 금지. 산문 의미를 해석하는 변조 금지(성격 반전·관계 조작·인과 뒤집기).
- 정답 불확실 문제 출제 금지 — 건전성 검증기를 통과한 문제만 사용자에게 도달한다.
- 카드에 없는 이름·수치·이미지 창작 금지. 변조 재료는 전부 카드 안의 실제 데이터 교환만.
- 문제당 광고·대기·재화 요소 금지. 시간 압박은 게임 규칙일 뿐 과금·강요 장치가 아니다.
- 앱 셸 전용 조건문 금지, 로어 회로 코드 수정 금지(재사용은 import만).

## 3. 완료 조건

1. `pnpm check` + 전체 테스트 + E2E + 경계 검사 통과.
2. 업로드(또는 월드컵 종료)→복구반 첫 문제 조작까지 10초 이내.
3. 같은 시드 → 같은 문제 세트·같은 결과 해시. 건전성 검증기 폐기 사유가 진단에 남는다.
4. 실카드 5종 기술 검증표 제출 (그룹화 정확도·폐기율·차단 정확도).
5. 커밋 분할: ①변조기+검증기 ②캐비닛 core ③react+개봉식 연결 ④검증. 푸시는 지휘자 검수 후.
6. 완료 보고: 사용 모델, 커밋 해시, 검증표, 스펙 이탈 사유. 이후 재미 검증(사용자 실플레이)은
   오너·지휘자가 진행하며 그 결과가 럭키★아카이브 착수 여부를 결정한다.
