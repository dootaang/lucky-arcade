# 럭키★오락실 백로그

> 우선순위 정본. 위에서 아래 순서로 진행하며 관문을 통과하기 전 다음 대형 게임을 시작하지 않는다.

## DONE — A1 오락실 입구와 이어하기 (2026-07-24)

사용자 화면에서 달라질 것:

- 첫 화면의 중심을 카드 업로드에서 `지금 바로 놀기`로 바꾼다.
- 저장된 플레이가 있으면 `이어하기`를 가장 먼저 보여준다.
- 게임을 `바로 한 판`, `반복 플레이`, `긴 게임`으로 나눠 고를 수 있게 한다.
- 카드 업로드는 `내 카드로 놀기`라는 별도 구역으로 이동한다.
- 캐비닛 이탈 시 마지막 진행과 최근 플레이 시간을 표시한다.

구현 항목:

1. `CabinetManifest`에 `sessionKind: instant | repeat | deep`, `launchKind: built-in | card`,
   `resumeLabel`, 예상 플레이 시간 범위를 추가한다.
2. 저장소에 공용 `RecentPlay` 조회 포트를 추가하되 게임별 상태는 각 캐비닛 스냅샷이 계속 소유한다.
3. 로비를 `이어하기 → 추천 내장 게임 → 내 카드로 놀기` 순서로 재배치한다.
4. 잔불 작전·월드컵·복구반·로어 회로를 같은 선언형 카탈로그에서 렌더링하고 앱의 장르별 분기를 제거한다.
5. 데스크톱·모바일에서 새 사용자 즉시 시작과 기존 사용자 한 번 클릭 복귀 E2E를 추가한다.

완료 관문:

- 카드가 0장인 새 사용자가 첫 화면에서 두 번 이하의 선택으로 내장 게임을 시작한다.
- 저장된 잔불 작전은 한 번의 선택으로 정확한 단계에 복귀한다.
- 개인 카드 업로드 기능과 기존 세 캐비닛 회귀가 유지된다.
- 초기 번들·버튼 반응 성능 예산을 넘지 않는다.

구현 결과: 카드 없는 첫 화면에 세 내장 게임을 먼저 배치했고, IndexedDB `RecentPlay`로 잔불 작전과
로어 회로의 최근 진행을 한 번에 이어 한다. 개인 카드 입구와 기존 게임은 아래 선택 구역으로 유지했다.
초기 JS gzip 105.0KiB, 데스크톱·모바일 E2E 통과.

## DONE — A2 하나의 콘텐츠팩, 여러 게임 (2026-07-24)

목표: 소녀전선 콘텐츠팩을 잔불 작전 전용 묶음에서 오락실 공용 카트리지로 승격한다.

1. 인물·표정·로어 참조를 담는 최소 `BuiltInContentPack` 계약을 사례에서 추출한다.
2. 기존 최애 월드컵 코어가 개인 카드 분석 결과와 내장 콘텐츠팩을 모두 입력받도록 어댑터를 둔다.
3. 같은 팩으로 60~120초짜리 스프라이트 순서 기억 게임 하나를 만든다.
4. 월드컵 결과에서 같은 세계의 짧은 게임 또는 잔불 작전으로 이동할 수 있게 한다.
5. 에셋 캐시는 캐비닛 사이에서 재사용하되 화면 이탈 후 Object URL·루프는 해제한다.

완료 관문:

- 같은 12명 데이터가 코드 복사 없이 최소 세 캐비닛(월드컵·짧은 게임·잔불 작전)을 연다.
- 각 캐비닛은 콘텐츠팩의 바이트나 LittleJS 타입을 직접 소유하지 않는다.
- 사용자가 같은 세계가 서로 다른 게임으로 바뀌었다는 것을 인물·표정으로 즉시 알아볼 수 있다.

구현 결과: `BuiltInContentPack` 계약과 소녀전선 어댑터를 만들고 같은 12명·표정 에셋으로
`소녀전선 최애 월드컵`, `작전 암호 기억`, `잔불 작전` 세 캐비닛을 연다. 월드컵 코어는 개인 카드와
내장팩이 같이 사용하며 새 기억 게임의 판정은 DOM 없는 순수 리듀서다.

## READY — A3 두 번째 봇카드 세계 온보딩

1. 사용·개조 범위가 확인된 두 번째 봇카드 하나를 선정한다.
2. 기존 파서와 콘텐츠 컴파일러로 NPC·로어·에셋 적합도 보고서를 만든다.
3. 새 게임 코어를 먼저 만들지 않고 기존 캐비닛 중 열 수 있는 것을 연결한다.
4. 부족한 계약은 두 콘텐츠팩에서 공통으로 필요한 경우에만 확장한다.

완료 관문:

- 앱 셸과 기존 캐비닛 코어를 복사하지 않고 두 번째 세계가 최소 두 캐비닛을 연다.
- 새 세계 추가가 선언·에셋·어댑터 중심의 국소 변경으로 끝난다.
- 두 세계를 로비에서 전환해도 저장·캐시·초기 로딩 예산이 유지된다.

기반 구현 결과: 공용 콘텐츠팩 스키마, 캐비닛 적합도 검사, `pnpm content:validate` CLI와 두 번째 세계
픽스처를 추가했다. 서로 다른 두 팩이 게임 코어 변경 없이 월드컵·기억 게임을 여는 단위 검증을 통과했다.
실제 두 번째 내장 세계의 배포는 **사용·개조 범위가 확인된 카드 선정**이 필요한 별도 콘텐츠 투입 단계다.
권리가 확인되지 않은 커뮤니티 카드는 임의로 포함하지 않는다.

## LATER — 반복 캐비닛 후보

우선 후보는 **랜덤 디펜스/서바이버**다. LittleJS를 재사용하고, 캐릭터 편성·로어 웨이브·배경을
콘텐츠팩이 공급한다. A1~A3 데이터가 나오기 전에는 구현하지 않는다.

승격 조건:

- 짧은 스프라이트 게임보다 반복 플레이 수요가 확인된다.
- 최소 두 콘텐츠팩에서 적·웨이브·플레이어 역할을 정직하게 매핑할 수 있다.
- 카드 데이터가 단순 스킨이 아니라 편성·웨이브·맵 중 두 가지 이상을 바꾼다.

후속 후보: 티어메이커, 세계관 골든벨, 로그라이크 탐험, 오토배틀러, 물리 퍼즐.

## 관찰할 제품 지표

- 새 사용자: 첫 게임 시작까지 걸린 선택 수와 시간.
- 기존 사용자: `이어하기` 사용률과 정확한 복귀 성공률.
- 캐비닛 간 이동: 한 세계에서 두 번째 게임으로 넘어간 비율.
- 세계 간 이동: 첫 세계 플레이 후 다른 콘텐츠팩을 연 비율.
- 중단 내성: 캐비닛 이탈·새로고침 뒤 진행 손실 건수.
- 재미 판단: 완료율만이 아니라 자발적 재시도와 다음 캐비닛 선택을 함께 본다.
