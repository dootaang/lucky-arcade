# SPEC-A2: 브라우저 카드 분석과 로어 회로 1호기

## 사용자가 얻는 결과

카드 파일을 넣으면 브라우저 안에서만 분석하고, 적합하면 즉시 `로어 회로`를 플레이한다. 시작 단어로 첫 기록을 발굴하고, 기록 안의 실제 키워드를 따라 목표 기록까지 간다. 카드에 없는 설정·문장·능력치는 만들지 않는다.

## 성능·개인정보 계약

- 파일 읽기와 분석은 Worker에서 수행한다. 새 파일을 넣으면 이전 Worker를 종료한다.
- `BlobBinarySource`는 요청된 구간만 읽는다. 대형 CharX를 메인 화면 메모리에 통째로 복제하지 않는다.
- 원본 카드, 카트리지, 플레이 기록은 IndexedDB에만 저장한다. 서버 업로드와 LLM 호출은 없다.
- 버튼 판정은 순수 reducer로 즉시 끝내고, 행동 영수증을 먼저 기록한 뒤 화면을 반영한다.
- 스냅샷은 750ms 지연 저장하며, 복구는 스냅샷 이후 행동만 재생한다.

## 게임 규칙

- 검증된 2~4단계 퍼즐 중 카드 지문과 시드로 하나를 결정한다.
- 시작 시 시작 로어 하나가 열린다. 열린 원문에 실제로 있던 키워드만 선택할 수 있다.
- 올바른 키워드는 그래프의 다음 한 층만 연다. 재귀 전체 공개는 하지 않는다.
- 목표 기록에 도착하면 승리한다. 잘못된 입력 3회면 탐사가 종료된다.
- 점수: `max(100, 1000 - 초과 이동×120 - 실수×150)`. 최단 이동은 감점하지 않는다.

## 모듈 경계

- `extract`: 카드에서 로어 회로 카트리지를 만든다.
- `cabinets/lore-circuit`: DOM 없는 상태·판정·점수·마이그레이션을 소유한다.
- `apps/web`: Worker, IndexedDB 어댑터, 화면 조립만 담당한다.
- React 없이 코어 테스트가 전부 실행돼야 한다.
