# SPEC-B1 — 개봉식 파이프라인 + 최애 월드컵 캐비닛

> 발주: 지휘자, 2026-07-23 (오너 전체 승인). 구현: Codex.
> 상위 문서: [README](../README.md) · [ROADMAP "제품 가설 v2"](./ROADMAP.md) · [UI-LAYOUT](./UI-LAYOUT.md).
> 북극성: **카드 업로드 후 30초 안에, 자기 카드의 인물·이미지·설정이 규칙에 반영됐음을 알아차린다.**

## 목표

카드 업로드 직후의 "개봉식" 흐름을 만들고, 그 1호 체험으로 **최애 이상형 월드컵**을 구현한다.
월드컵은 대표 게임이 아니라 ①개봉식 ②NPC 그룹화 파이프라인의 실전 검증 ③결과 자랑 카드라는
세 역할만 가진다.

## 1. 작업 범위 (건드릴 파일)

기존 구조를 그대로 사용한다: `packages/extract`(npc.ts·report.ts), `packages/engine`,
`packages/cabinet-sdk`, `packages/ui`, `apps/web`(features/·workers/·routes/), `cabinets/`.

### B1-a. 개봉식 라우팅 (apps/web)

- 카드 업로드 완료 → 기존 Worker 분석 결과(재료 검사)로 **첫 체험을 자동 선택**:
  1. 월드컵 관문 통과 → 최애 월드컵 개봉식
  2. 관문 미달 + 복구반 재료 충족 → (SPEC-B2 완성 전까지) 적합도 요약 화면
  3. 둘 다 미달 → "이 카드로 가능한 놀이" 안내 (억지 활성화 금지)
- 업로드→첫 상호작용 가능까지 **10~30초** 이내. 대형 카드는 분석 완료 전에라도
  NPC 그룹이 준비되는 즉시 월드컵을 시작할 수 있게 점진 로딩을 허용한다.
- **로어 회로 강등**: 홈/내비의 로어 회로 진입점을 `실험실` 섹션으로 이동한다. 기능 삭제 금지.

### B1-b. 월드컵 관문 (packages/extract 확장)

- 셀렉터 `favoriteCupEligibility(cartridge)` 신설:
  - 서로 다른 NPC(동일 인물의 감정·의상 변형은 1명으로 통합) **8명 이상**
  - NPC 그룹 confidence가 기준치 이상인 인물만 후보로 인정 (기준치는 상수로 두고 보고서에 노출)
  - 배경·아이콘·기타(`기타` 폴백) 이미지는 후보 제외
  - 각 인물의 대표 초상화 1장 선택 규칙(기본: default/natural 계열 우선)을 결정론으로 정의
- 결과는 기존 `Sourced<T>` 계약(value/source/confidence/evidence)으로 반환하고
  적합도 보고서(report.ts)에 "월드컵: 가능/불가+사유"를 추가한다.

### B1-c. 월드컵 캐비닛 (cabinets/favorite-cup — core/react 분리)

- `core`(순수, DOM 금지): 후보 선정(8/16강 자동 — 후보 9~15명이면 8강, 16명 이상이면 16강),
  대진 셔플은 `packages/engine` 시드 RNG 사용(시드=날짜+카드 지문 기본, 재대진 허용),
  진행 상태 리듀서(선택→다음 매치→우승), 입력 기록·결과 해시 기록.
- `react`: 두 초상화 대결 화면(탭/클릭 즉시 다음 매치, 애니메이션은 prefers-reduced-motion 존중),
  진행도 표시, 우승 화면.
- cabinet-sdk 매니페스트로 등록. **앱 셸에 월드컵 전용 조건문 배선 금지.**

### B1-d. 결과 화면 + 자랑 카드

- 우승자 + 4강을 "최애 라인업"으로 표시 (점수보다 인물 먼저 원칙).
- "이 카드의 분석 결과 보기"(적합도 요약)와 "다른 놀이 하러 가기" 연결.
- **자랑 카드 이미지 로컬 생성**: 우승자 초상·이름·카드 제목·앱 워터마크를 담은 이미지를
  캔버스로 생성해 사용자가 직접 저장. 자동 업로드·자동 공유 금지.
  저장 버튼 옆에 "공유 전 카드 이용 규정을 확인하세요" 1줄 안내.
- 도전 코드(공유용 텍스트)에는 시드·입력·카드 지문만. 이미지·원문 포함 금지.

### B1-e. 검증

- core 단위 테스트: 후보 선정 결정론, 통합 규칙(감정/의상 1명), 8/16강 편성, 시드 재현,
  관문 차단(7명 카드에서 비활성).
- 브라우저 E2E: 합성 카드로 업로드→개봉식→월드컵 완주→자랑 카드 저장 여정 1건,
  관문 미달 카드의 폴백 여정 1건.
- 실카드 검증: 보유 실카드 중 NPC 다수 카드 2종 + 미달 카드 1종에서 수동 확인 기록.

## 2. 금지사항

- LLM 호출·네트워크 전송(카드·이미지·결과의 외부 업로드) 금지. 모든 처리는 로컬.
- **티어메이커 구현 금지** (백로그 — 유저 요청 실측 후 승격).
- 카드에 없는 인물·설정 생성 금지. NPC 후보를 휴리스틱 추측으로 보정하지 말 것(관문 미달이면 비활성이 정답).
- 카드 원문·에셋 바이트의 재수출 경로 신설 금지. 자랑 카드는 사용자 수동 저장만.
- 앱 셸에 캐비닛 전용 조건문 금지 (매니페스트 등록만). ROADMAP 모듈화·성능 계약 준수.
- 기존 로어 회로·보고서 기능 삭제 금지 (실험실 이동만).

## 3. 완료 조건

1. `pnpm check` + 전체 테스트 + E2E 통과, 경계 검사(check-boundaries) 위반 0.
2. 관문 통과 카드: 업로드→첫 매치 조작까지 30초 이내(로컬 기준 실측 기록).
3. 관문 미달 카드: 월드컵이 보이지 않고 폴백 안내가 뜬다.
4. 같은 시드·같은 입력 → 같은 대진·같은 결과 해시.
5. 커밋 분할: ①관문·extract ②캐비닛 core ③react+개봉식 라우팅 ④검증. 푸시는 지휘자 검수 후.
6. 완료 보고: 사용 모델, 커밋 해시, 실카드 3종 결과(후보 수·관문 판정·완주 여부), 스펙 이탈 사유.
