# 럭키★오락실 로드맵

> 합의 기준: 2026-07-23 지휘자(Claude)·코덱스 교차 검수 완료본.
> 원칙·라이선스·콘텐츠 권리는 [README](../README.md), UI 기준은 [UI-LAYOUT](./UI-LAYOUT.md).

## 구조

```
봇카드
  → 카드 읽기 어댑터 (simbot에서 순수 함수만 선별 이식, 출처 기록)
  → 오락실 공용 데이터 (로어 그래프 · NPC/스프라이트 그룹 · 경제표 · 카드 지문)
  → 재료 검사기 (캐비닛 활성화 판정 + 적합도 리포트)
  → 소형 판정 코어 (시드 난수 · 입력 기록 · 결과 해시 · 게임 버전)
  → 캐비닛들 (표현 자유: shadcn/React 기본, 캐비닛별 지연 로딩)
```

- 모든 추출 결과는 `{ value, source, confidence, evidence, derived }` 공통 계약을 따른다.
- 판정·점수·정답은 항상 판정 코어가 소유한다. 렌더링 라이브러리는 표현·입력만 담당한다.

## 모듈화·무이주 계약 (2026-07-23 오너 조건 — ★강제)

패키지 구조와 의존 방향을 처음부터 다음으로 고정한다. 목적: 나중에 갈아엎는 이주(JS→TS,
저장소 교체, 프레임워크 이주)가 구조적으로 불가능하거나 국소 수술로 끝나게 한다.

```
packages/contracts    버전 스키마·타입·마이그레이션 (도메인 의존 0)
packages/card-io      BinarySource 기반 카드 파싱·에셋 지연 색인
packages/extract      카드→놀이 재료 (로어·NPC·경제표·적합도)
packages/engine       소형 판정 코어 (시드·입력·해시, DOM 금지)
packages/persistence  행동 WAL·스냅샷·에셋 캐시 포트
packages/cabinet-sdk  캐비닛 매니페스트·생명주기 계약
packages/ui           shadcn·디자인 토큰·공용 React 부품
cabinets/*/core       게임별 순수 리듀서 (DOM 금지)
cabinets/*/react      게임별 화면·입력·선택형 렌더러
apps/web              Vite React 앱 — 선언형 캐비닛 조립
apps/report-cli       적합도 측정기 (extract 재사용, UI 무관)
```

1. **의존 방향 단방향**: card-io→contracts, extract→card-io/contracts,
   cabinet core→contracts/engine/cabinet-sdk, cabinet react→자기 core/ui/React, apps→모든 조립 계층.
   캐비닛 core는 extract 구현을 호출하지 않고 계약 데이터만 받는다. React/DOM import는
   `packages/ui`, `cabinets/*/react`, `apps/web` 밖에서 금지한다. 위반은 CI 실패한다.
2. **TypeScript strict를 첫 커밋부터**, 예외 없음. "타입만 .ts, 로직은 .js" 금지.
3. **코어 테스트는 Node에서 브라우저 없이 실행** — cabinet core/engine/extract에 jsdom 의존이 생기면 설계 위반.
4. **저장은 용도별 포트 뒤에 둔다** — `ActionLogStore`, `SnapshotStore`, `AssetCache`,
   `CardSourceStore`를 분리한다. 큰 스냅샷·이미지 작업이 작은 행동 영수증을 막지 않아야 한다.
5. **캐비닛 추가 = 독립 폴더 + 선언형 매니페스트** — 앱 셸에 장르별 조건문이나 전용 배선을
   추가하지 않는다. 빌드 등록은 매니페스트 또는 자동 생성 등록부만 사용한다.
   캐비닛 계약: 필요 재료·최소 데이터 수·요구 신뢰도·시드 규칙·입력 기록·점수 계산·
   공유 가능 정보·카드 원문 공개 범위 선언.
6. **외부 렌더링 라이브러리(Phaser/KAPLAY/Matter/Cytoscape)는 해당 캐비닛의 `react` 계층 + dynamic import로 격리.**
7. 패키지 간 참조는 workspace protocol + `exports` 공개 API만. 내부 파일 직접 import 금지.

## 성능·저장 계약 (럭키 시뮬레이터 장기 회차 교훈 — ★강제)

- 카드 파일 공용 입구는 `BinarySource { size, read(offset,length) }`다. 전체 파일을 공용 상태의
  거대한 `Uint8Array`로 보관하지 않는다. 에셋은 메타데이터와 지연 참조를 우선 반환한다.
- 카드 해제·그래프 분석·썸네일 생성은 Worker 경계를 전제로 JSON/구조화 복제 가능한 DTO만 사용한다.
  게임 판정 전체 Worker 이전은 실측이 8~10ms 관문을 넘을 때만 시행한다.
- 행동은 공용 직렬 대기열에서 판정하고 소형 WAL을 먼저 확정한다. 전체 스냅샷·통계·캐시 정리는
  유휴 차선에서 병합하며, 과거 전체를 매 행동마다 직렬화·재해시하지 않는다.
- 목록은 최대 192px WebP 썸네일을 사용한다. 화면 근처만 로딩하고 낡은 요청은 취소한다.
  Object URL·메모리·영구 캐시는 모두 상한과 퇴출 시 해제 규칙을 가진다.
- 완료 관문: 버튼 눌림 표시 p95≤50ms(데스크톱), 준비된 로컬 행동 결과 p95≤100ms,
  50→1,000행동 p95 성장≤1.2배, 50ms 초과 메인 스레드 장기 작업 없음, 대형 목록 전체 DOM 생성 금지.
- 성능 진단은 버튼 페인트·대기열·판정·WAL·상태 반영·결과 페인트·백그라운드 저장을 분리 계측한다.

3. **공용 데이터 계약** — Zod 런타임 스키마, source/confidence/evidence/derived, 버전 계약. ✅ 2026-07-23
4. **적합도 측정기** — 게임 UI 없이 카드별 보고서: 로어 노드·연결, SCC 압축 깊이·도달률· ✅ 2026-07-23
   최단거리 지름·고립률, 생성 가능 퍼즐 수, NPC 그룹 정확도 후보, 스프라이트 그룹 수,
   경제표 존재 여부, 가능한 캐비닛과 불가 사유.
5. **로어 회로 go/no-go** — ✅ GO, 2026-07-23. CharX·PNG 실보유 카드 13종을 측정해
   6종이 검증된 2~4단계 연쇄 최소 3개 관문을 통과했고 7종은 미통과했다. 각 퍼즐은 실제
   `activateLore`를 시드 고정으로 재실행해 검증했다. 따라서 로어 회로를 1호 캐비닛으로 확정하되,
   재료 검사 통과 카드에서만 활성화한다. “모든 카드 범용” 주장은 폐기한다.
6. **로어 회로 발굴 모드 MVP** — ✅ 2026-07-23. 브라우저 Worker 분석, 로컬 카드 보관함,
   검증 퍼즐 카트리지, 한 층씩 발굴하는 순수 판정 코어, 행동 WAL·스냅샷 복구,
   데스크톱·모바일 플레이 화면을 연결했다. 합성 카드 E2E와 실카드 3종 추가 점검을 통과했다.
   다음 확장은 실제 사용자 플레이 피드백 후 최단경로 도전·과부하·잠입 순서로 연다.

## 그래프 지표 정의 (합의 확정)

- 최장 단순 경로는 쓰지 않는다(순환 그래프에서 부정확). **SCC 압축 후 DAG 깊이 + 최단거리 지름 +
  최소 입력 횟수 BFS 검증**을 쓴다.
- 그래프는 2층: **잠재 그래프**(구조적 연결 가능성)와 **실행 그래프**(특정 시드·턴에서 실제
  activateLore가 활성화한 관계). 퍼즐 정답 검증은 실행 그래프 재실행으로 한다.
- 정규식 키 로어는 기본 퍼즐에서 제외(사람이 읽을 발굴 단어가 없음), 고급 규칙으로 격리.

## 제품 가설 v2 — 실험 계획 (2026-07-23 오너 전체 승인, 현행 기준)

지휘자·코덱스 교차 검수 + 한국 게임 친숙도 조사 2건(교차 검증 완료)으로 확정.
아래는 "확정 제품"이 아니라 **검증할 가설**이며, 다음 피벗은 의견이 아니라 실제 플레이
데이터만 일으킬 수 있다.

- **북극성(경험 정의)**: 카드 업로드 후 30초 안에, 사용자가 자기 카드의 실제 인물·이미지·
  설정이 플레이 규칙에 반영됐음을 알아차린다.
- **시간 예산**(2025 게임이용자 실태조사 근거 — 이용률 50.2%, 시간 부족이 최대 이탈 사유):
  첫 재미 10~30초, 사건 20~60초, 한 판 3~5분. 15분 플레이는 사용자가 원할 때만 확장.
- **역할 배치**:
  - 입구·공유 실험: **최애 월드컵** (개봉식 + 결과 자랑 카드. NPC 관문 미달 카드에서는 자동 비활성)
  - 첫 게임 실험: **카드 복구반** (기계적 변조 찾기 — "이상한 곳을 눌러라")
  - 대표작 가설: **럭키★아카이브** (경로 선택→30초 사건→기록 복원→탈출/심화 4동작만.
    덱빌딩·전투·강화는 검증 전 반입 금지. 상세 설계는 실험 결과 전까지 동결)
  - 로어 회로: 사용자 메뉴에서 **실험실로 강등** (판정기·그래프·Worker 분석은 아카이브·복구반의 기반으로 재사용)
  - 럭키★러시: 프레임 폐기, 검증된 사건 아이디어만 아카이브 방 사건 후보로 보존
- **착수 순서(동결)**: ①공용 카드 재료 검사기 브라우저 연결 → ②최애 월드컵 개봉식 →
  ③카드 복구반 → ④실카드 5종 기술 검증 + 실플레이 재미 검증 → ⑤데이터 확보 후에만 아카이브 방 하나
- **구현 현황 (2026-07-23)**: ①~③ 구현 완료. ④의 기술 검증은 실카드 5종에서 통과
  ([측정 보고서](../reports/npc-accuracy-2026-07-23.md)); 다음 관문은 오너의 실플레이 재미 검증이다.
  이 검증 전에는 럭키★아카이브 상세 설계·티어메이커·추가 캐비닛을 착수하지 않는다.
- **검증 2단계 분리**: 기술 검증(NPC 그룹화 정확도·정답 생성 건전성·관문 차단 정확도) /
  재미 검증(설명 없이 시작, 재도전, 다른 카드 투입 — 실제 행동으로 측정. 아카라이브 게시 반응은
  확산성 지표로만 사용)
- **결과 화면 원칙**: 점수보다 인물 먼저 ("가장 많이 복원한 인물", "우승 최애").
- **순화 번역 원칙**: 방치 성장→탐험 기록 자동 정리, 강화→업적 기반 복원 등급(운 아님),
  뽑기→플레이로 발견한 것의 개봉 연출(중복 없는 확정 신규), 희귀도→카드 내부에서 실제로 드문
  에셋·고립 로어(earned rarity), 일일 과제→고정 시드 도전(접속 강요 없음).
- **리텐션**: "새 카드를 받으면 넣어본다"는 검증된 구조가 아니라 **잠재 경로(가설)**로 취급하고
  재방문 시 신규 카드 투입 비율을 실측한다.

## 캐비닛 우선순위 (가설 v2 기준)

| 순위 | 캐비닛 | 관문 |
|---|---|---|
| 1 | 최애 월드컵 (개봉식·공유) | 서로 다른 NPC ≥ 8명, 그룹 confidence 기준, 동일 인물 감정/의상 통합 |
| 2 | 카드 복구반 (첫 게임 실험 — 舊 NPC 검문소 계승) | NPC 그룹 정확도 관문 + 변조 유일해 검증 |
| 3 | 럭키★아카이브 (대표작 가설) | 월드컵·복구반 실측 데이터 확보 후 착수 |
| 백로그 | 티어메이커(월드컵 파이프라인 재사용 — 유저가 원할 때 승격), 디펜스(기록 붕괴), 스프라이트 파티, 세계관 탐정(golden bell 연출, explicitSpeaker 한정), 잔해 발굴·시장 퍼즐, 오토배틀러·올림픽 | 각 항목별 기존 관문 유지 |
| 실험실 | 로어 회로(강등), 혼돈 리그, 카드 해부학 로그라이크, 자동 원정 방치형, 카드 제작자용 검증 게임 | MVP 밖 |

## 공유 계약

모든 캐비닛 공통: 오늘의 시드(날짜+카드 지문), 개인 최고 기록, 입력 리플레이, 캐비닛 버전, 도감·업적.
공유 코드에는 시드·입력·점수·카드 지문만 담는다. 서버 없이 로컬 재현이 정본이며,
점수판 서버는 나중에 얹어도 게임 정본은 로컬 엔진이다.

## 렌더링 라이브러리 조달 (합의)

- 기본: shadcn/React + CSS. 로어 회로 입문 화면 포함.
- 전체 로어망·별자리 도감 화면: 자체 SVG vs Cytoscape.js(MIT) **비교 시제품 후 결정**. 판정 연산은 항상 자체.
- 플링코: Matter.js(MIT) — 물리는 연출 전용, 낙하 결과는 판정 코어가 선결정.
- 액션 스낵: KAPLAY 3001 안정판 고정(v4000 알파 금지). 본격 아케이드: Phaser 4 캐비닛별 동적 로딩.
- 로그라이크 승격 시: rot.js(BSD). boardgame.io는 배포 장기 정체·기능 중복으로 미채택.
