# 럭키★오락실 로드맵

> 합의 기준: 2026-07-23 지휘자(Claude)·코덱스 교차 검수 완료본.
> 원칙·라이선스·콘텐츠 권리는 [README](../README.md), UI 기준은 [UI-LAYOUT](./UI-LAYOUT.md).

## 구조

```
봇카드
  → 카드 읽기 어댑터 (simbot에서 순수 함수만 선별 이식, 출처 기록)
  → 오락실 공용 데이터 (로어 그래프 · NPC/스프라이트 그룹 · 경제표 · 카드 지문)
  → 재료 검사기 (캐비닛 활성화 판정 + 적합도 리포트)
  → 소형 판정 코어 (시드 난수 · 입력 기록 · 결과 해시 · 게임 버전)
  → 캐비닛들 (표현 자유: shadcn/React 기본, 캐비닛별 지연 로딩)
```

- 모든 추출 결과는 `{ value, source, confidence, evidence, derived }` 공통 계약을 따른다.
- 판정·점수·정답은 항상 판정 코어가 소유한다. 렌더링 라이브러리는 표현·입력만 담당한다.

## 단계 (합의된 첫 지시 범위 = 1~5)

1. **저장소 기초** — GPL-3.0-or-later, 프로그램/카드 권리 분리, PROVENANCE, 원칙 문서,
   React+Vite+TypeScript+Tailwind+shadcn 툴체인. ✅ 2026-07-23 (이 커밋)
2. **코드 이식 경계** — 카드 파싱·로어 정규화/활성화·카드 지문의 순수 함수만 선별 이식.
   UI·PlaySession·LLM·대형 저장 엔진 이식 금지. 출처 문서 기록.
3. **공용 데이터 계약** — 전 추출물에 value/source/confidence/evidence/derived.
4. **적합도 측정기** — 게임 UI 없이 카드별 보고서: 로어 노드·연결, SCC 압축 깊이·도달률·
   최단거리 지름·고립률, 생성 가능 퍼즐 수, NPC 그룹 정확도 후보, 스프라이트 그룹 수,
   경제표 존재 여부, 가능한 캐비닛과 불가 사유.
5. **로어 회로 go/no-go** — 유형별 실카드 표본(단순·로어 대형·모듈 결합·확률/재귀·에셋 대형)으로
   측정. 각 퍼즐을 실제 `activateLore` 재실행으로 자동 검증. 통과 시에만 1호 캐비닛 UI 착수.

## 그래프 지표 정의 (합의 확정)

- 최장 단순 경로는 쓰지 않는다(순환 그래프에서 부정확). **SCC 압축 후 DAG 깊이 + 최단거리 지름 +
  최소 입력 횟수 BFS 검증**을 쓴다.
- 그래프는 2층: **잠재 그래프**(구조적 연결 가능성)와 **실행 그래프**(특정 시드·턴에서 실제
  activateLore가 활성화한 관계). 퍼즐 정답 검증은 실행 그래프 재실행으로 한다.
- 정규식 키 로어는 기본 퍼즐에서 제외(사람이 읽을 발굴 단어가 없음), 고급 규칙으로 격리.

## 캐비닛 우선순위

| 순위 | 캐비닛 | 관문 |
|---|---|---|
| 1 | 로어 회로 (발굴→최단경로→과부하→잠입, 한 성장 구조) | 5단계 적합도 실측 통과 |
| 2 | NPC 검문소 (Papers, Please식 데이터 대조) | NPC 그룹 정확도 관문 (오합침률·감정/의상 분리율·confidence) |
| 3 | 스프라이트 파티 (표정 맞히기·몽타주·분류·다른 하나·순서 기억 통합) + 세계관 탐정 (원문 대조 퀴즈, 화자 퀴즈는 explicitSpeaker 카드 한정) | — |
| 4 | 잔해 발굴(확률 발굴 스낵)·시장 퍼즐 | 무과금·무광고·확률 공개·시드 재현 금지선 / 경제표 출처 검증 |
| 5 | 오토배틀러 리그·카드 올림픽 데일리 | 명시 능력치 카드만 |
| 실험실 | 혼돈 리그(카드 파일 통계 배틀), 카드 해부학 로그라이크, 자동 원정 방치형, 카드 제작자용 검증 게임 | MVP 밖 백로그 |

## 공유 계약

모든 캐비닛 공통: 오늘의 시드(날짜+카드 지문), 개인 최고 기록, 입력 리플레이, 캐비닛 버전, 도감·업적.
공유 코드에는 시드·입력·점수·카드 지문만 담는다. 서버 없이 로컬 재현이 정본이며,
점수판 서버는 나중에 얹어도 게임 정본은 로컬 엔진이다.

## 렌더링 라이브러리 조달 (합의)

- 기본: shadcn/React + CSS. 로어 회로 입문 화면 포함.
- 전체 로어망·별자리 도감 화면: 자체 SVG vs Cytoscape.js(MIT) **비교 시제품 후 결정**. 판정 연산은 항상 자체.
- 플링코: Matter.js(MIT) — 물리는 연출 전용, 낙하 결과는 판정 코어가 선결정.
- 액션 스낵: KAPLAY 3001 안정판 고정(v4000 알파 금지). 본격 아케이드: Phaser 4 캐비닛별 동적 로딩.
- 로그라이크 승격 시: rot.js(BSD). boardgame.io는 배포 장기 정체·기능 중복으로 미채택.
