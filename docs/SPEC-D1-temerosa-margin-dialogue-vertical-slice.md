# SPEC-D1 — 테메로세: 여백, 대사 중심 JRPG 수직 시제품

> 상태: 인기투표 반영 v2 설계 정본. 이 문서와 `TEMEROSA-DIALOGUE-BIBLE-V2.md`를 승인 없이 분리 구현하지 않는다.
> 콘텐츠 원본: `C:\freetalk\테메로세\Temerosa_Finale.charx` 외 테메로세 시리즈 3종.

## 1. 플레이어가 실제로 경험할 것

플레이어는 무너진 피쿼드 폐허에서 박니은의 원격 연결로 깨어난다. 현재의 니은은 세계 끝에서 사상 지평을 유지해
대멸종을 막고 있으므로 현장 동료처럼 이동하지 않는다. 대신 첫 임무를 주도하고, 플레이어의 선택과 죽음을 연결
너머에서 관찰한다. 알제가 폐허의 피쿼드 계약 단말기를 열고, 플레이어는 페일·카노·네모/바치칼 가운데 두 명과
과거를 현실에 덮어쓰는 트레인헤드를 추적한다. 첫 사망 또는 보스전 종료 뒤에야 워어즈와 여백을 발견한다.

이 시제품의 목적은 다음 한 문장을 검증하는 것이다.

> 전투를 다시 시작하는 이유가 보상뿐 아니라 “이번 조합에서는 이 인물이 뭐라고 할까?”가 되는가.

## 2. 장르와 표현

- 장르: 일러스트 기반 탐험형 덱빌딩 JRPG + 로그라이트.
- 한 회차 목표: 15~25분. 언제든 중단 저장하고 돌아올 수 있다.
- 메뉴·덱·대화 기록·선택지: React.
- 전투·배경 전환·컷인·파티클·보스 왜곡: Phaser 4 지연 로딩.
- 판정·시드·대사 선택·결과 해시: DOM 없는 순수 TypeScript 코어.
- 런타임 LLM 호출: 없음. 모든 발화는 검수된 대사집에서 선택한다.
- 원본 일러스트는 도트 이동 스프라이트로 가장하지 않는다. 인물 컷인, 카드, 반신 초상, 배경, 괴물 전면 일러스트로 사용한다.

## 3. 첫 수직 시제품 범위

### 포함

- 허브 2개: 피쿼드 폐허 임시 거점, 첫 사망/보스 종료 뒤 열리는 여백.
- 지역 1개: 피쿼드 폐허와 과거 메트로 정거장을 잇는 7노드 경로.
- 동료 3명 중 2명 선택: 페일, 카노, 네모/바치칼.
- 임무 주도자: 박니은. 현지 연결자: 알제. 부활 담당: 워어즈.
- 과거 기록 인물: 라일라/리엘.
- 일반 적 3종과 환경 위협: 토끼벌, 눈눈늑대, 신경나무, 과거 회귀 충돌.
- 보스 1종: 트레인헤드.
- 기본 전투 덱 12장 + 동료별 전용 카드 4장씩.
- 수치: 생명력, 인간성, 짐승성. 배고픔·갈증은 D1에서는 사건 조건으로만 사용하고 상시 감소시키지 않는다.
- 사건 3개, 휴식 1개, 전투 3개, 보스 1개.
- 사망·부활·중단 저장·재개.
- 핵심 7인 대사 성경·관계 규칙과 첫 회차 전체 대본 v2.

### 제외

- 세 지역 전체, 10개 세력 전체, 30인 플레이어블화.
- 실시간 자유 이동, 타일맵, 포켓몬식 포획.
- 음성 더빙, BGM. D1은 합성 효과음과 화면 연출만 허용한다.
- 생성형 대사, 사용자 자유입력에 대한 의미 응답.
- 공용 재화, 과금, 서버 점수판.
- 성인 에셋과 성적 설정. 콘텐츠 컴파일 단계에서 허용 목록 밖의 항목을 제외한다.

항상 위 순서를 보장하고, 두 번째부터 두 노드의 위치와 적 구성을 바꾼다.

## 5. 대사가 장식이 아니게 하는 규칙

1. **정보**: 위험, 보스 규칙, 자원 변화 중 최소 하나를 말 또는 표정으로 알려준다.
2. **관계**: 같은 사건도 동료 조합에 따라 의견과 후속 효과가 달라진다.
3. **기억**: 사망 횟수, 같은 보스 연속 패배, 이전 선택, 동료 동행 횟수를 기억한다.
4. **대가**: 중요한 대화 선택은 인간성·짐승성·관계·카드 중 최소 하나를 바꾼다.
5. **침묵**: 모든 행동에 말하지 않는다. 긴장 장면에서는 의도적으로 무음 구간을 둔다.
6. **재현**: 같은 게임 버전·시드·입력 기록이면 같은 발화가 선택된다.

## 6. 대사 코어 계약

```ts
type DialogueIntent =
  | "scene" | "exploration" | "combat-start" | "skill" | "hurt"
  | "low-humanity" | "high-beasthood" | "victory" | "defeat"
  | "revive" | "banter" | "choice" | "boss-hint";

type DialogueCondition = {
  sceneIds?: string[];
  speakerIds?: string[];
  companionIds?: string[];
  enemyIds?: string[];
  minDeaths?: number;
  maxDeaths?: number;
  minHumanity?: number;
  maxHumanity?: number;
  minBeasthood?: number;
  requiredFlags?: string[];
  forbiddenFlags?: string[];
};

type DialogueLine = {
  id: string;
  speakerId: string;
  intent: DialogueIntent;
  text: string;
  portrait: string;
  priority: 0 | 1 | 2 | 3;
  cooldown: number;
  oncePerRun?: boolean;
  onceEver?: boolean;
  condition: DialogueCondition;
  effects?: GameEffect[];
};

type DialogueCue = {
  intent: DialogueIntent;
  sceneId: string;
  actorId?: string;
  targetId?: string;
  enemyId?: string;
  priority: 0 | 1 | 2 | 3;
};
```

- 전투 코어는 문자열을 직접 선택하지 않고 `DialogueCue`만 발생시킨다.
- 대사 감독은 현재 `RunState`, 영구 `StoryMemory`, 최근 발화 기록을 받아 후보를 걸러낸다.
- 후보 정렬은 우선순위 → 미사용 가중치 → 결정론 난수 순이다.
- 발화 효과는 별도 `GameEffect`로 판정한다. UI가 대사를 표시했다고 직접 상태를 변경하지 않는다.
- 저장에는 대사 ID, 선택지 ID, 발생 턴, 결과 효과만 기록한다. 전체 텍스트는 버전 콘텐츠에서 복원한다.

## 7. 반복·겹침 방지

- 같은 대사 ID는 기본 10회 발화 동안 재선택 금지.
- 같은 의도·같은 화자 조합은 2회 연속 금지.
- 전투 일반 대사는 한 라운드 최대 1개, 보스 힌트와 사망 대사는 예외.
- 우선순위 3의 이야기 대사가 재생 중이면 낮은 우선순위 대사는 폐기한다.
- 우선순위 2 이상끼리 충돌하면 큐에 최대 2개만 보관한다.
- 4초 안에 읽지 못할 길이의 전투 대사는 자동으로 전투를 멈추지 않는다. 핵심 장면은 명시적으로 일시정지한다.
- 이미 알아낸 보스 힌트는 장문 재설명 대신 짧은 확인 대사로 교체한다.

## 8. 관계와 사상

D1 관계치는 화면에 숫자로 노출하지 않는다. `낯섦 → 익숙함 → 신뢰 → 균열/유대` 상태로만 표현한다.

- 페일: 결속. 관계와 이름을 부여함으로써 짐승이 단순한 본능을 넘어선다고 믿는다.
- 카노: 화답. 숭배받는 존재라면 실제로 응답해야 한다고 믿지만 신의 역할 뒤에 두려움을 숨긴다.
- 네모/바치칼: 결속. 과거를 없애지 않고 이름과 책임을 다시 받아들이려 한다.
- 박니은: 이해 불가능성을 믿지만, 이해하지 못하는 세계를 혼자 지키는 모순을 산다.
- 알제: 책임을 피하고 싶지만 옛 시스템과 생존자를 방치하지 못한다.
- 워어즈: 관찰자. 플레이어에게 정답을 주지 않고 선택이 쌓이는 모습을 본다.

선택이 특정 사상 점수를 올려 엔딩을 자동 결정해서는 안 된다. 사상 기록은 동료 반응과 보스 질문의 문맥을 바꾸며,
마지막 판단은 플레이어가 직접 선택한다.

## 9. 성능·에셋 선행 관문

테메로세 카드군은 메타데이터의 이미지 형식과 실제 페이로드가 다르다. D1 콘텐츠 컴파일 전에 다음을 해결한다.

1. 확장자 선언이 아니라 실제 바이트 시그니처로 PNG/JPEG/WebP를 판별한다.
2. 원본 941개를 한 번에 번들하지 않고 D1 허용 목록만 별도 콘텐츠팩으로 만든다.
3. 로비에는 테메로세 에셋과 Phaser를 넣지 않는다. 캐비닛 진입 뒤 필요한 장면 단위로 로드한다.
4. 초상은 192px·384px 파생본, 전투 컷인은 화면 크기별 파생본을 사용한다.
5. 장면 이탈 시 Object URL, 텍스처, 오디오, 입력, Phaser Scene을 모두 해제한다.
6. 이름에 `naked`, `masturbation`, 성인 상태가 포함된 항목과 별도 안전 목록에 없는 에셋은 컴파일 실패로 처리한다.

## 11. 완료 관문

- 첫 입력 뒤 30초 안에 박니은이 등장하고 5분 안에 피쿼드·메트로·인기 동료 중 둘을 확인한다.
- 박니은은 현장 파티원이 되지 않고 사상 지평을 유지한다. 이동시키는 연출은 설정 위반으로 실패다.
- 세 동료 조합 모두 첫 회차를 시작부터 귀환까지 완주한다.
- 첫 사망 전에는 워어즈가 장황한 세계 설명을 하지 않는다. 승리 무사망 경로에서도 보스 종료 뒤 여백을 발견한다.
- 라일라 기록은 현재의 플레이어를 확실히 인식했다고 단정하지 않으며, 과거 기록만으로 Finale의 결론을 대신 말하지 않는다.
- 승리·사망·중단 후 재개가 모두 같은 이야기 상태를 복원한다.
- 동일 시드·입력 로그의 결과 해시와 대사 ID 열이 동일하다.
- 한 회차에서 동일 일반 대사가 반복되지 않는다.
- 트레인헤드 규칙은 대사 없이도 판정 가능하고, 대사는 정답을 대신 누르지 않고 이해를 돕는다.
- D1 콘텐츠팩에 금지 에셋이 0개다.
- 로비 초기 JS 200KiB gzip 예산을 유지하고 Phaser·테메로세 에셋은 지연 청크다.
- 사용자에게 보이는 버튼 반응 p95 50ms, 준비된 장면 전환 p95 300ms 목표를 지킨다.
- 오너가 다른 동료 조합으로 즉시 재도전할 의사가 있다고 판정한다.
